### Plugin System Risk

The plugin system (`internal/plugin/`) uses Go's `plugin.Open()` to load `.so` shared objects at runtime. This is an inherent trust boundary:
- Plugins run with full process privileges
- No sandboxing or capability restriction
- Mitigated by file-system-level access control on the plugin directory

**Recommendation:** Document plugin security model. Consider adding code signing verification for plugin `.so` files.
